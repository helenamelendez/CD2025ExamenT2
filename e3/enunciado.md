## Ejercicio 3: Diagrama de casos de uso

Una pequeña biblioteca quiere informatizar un sistema sencillo para gestionar el préstamo de libros.

En el sistema participan dos tipos de usuarios:

- Usuario: puede buscar libros disponibles y solicitar el préstamo de un libro.

- Bibliotecario: se encarga de registrar los préstamos y registrar la devolución de los libros.

El sistema debe permitir las siguientes acciones:

- Buscar libros en el catálogo.

- Solicitar préstamo de un libro.

- Registrar préstamo.

- Registrar devolución.

A partir de la descripción anterior:

1. Identifica los actores del sistema.

2. Identifica los casos de uso.

3. Dibuja un diagrama de casos de uso UML que represente el sistema.

4. Añade las explicaciones y el diagrama creado en `docs/explicacion.md`.

5. Adjunta una captura del diagrama en `docs`.

6. Usa el siguiente formato para los diagramas:

```mermaid
graph LR

Actor[Actor]

Acción((Acción))

Actor --> Acción

```
